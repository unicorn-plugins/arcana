# 아르카나 AI 팀원 역량 가이드

> 우리 팀이 무엇을 할 수 있는지, 누구에게 무엇을 맡기면 되는지 한눈에 보는 안내서입니다.

---

## 1. 역할별 산출물

각 팀원이 만들어낼 수 있는 결과물을 정리했습니다.

### 주디 (PD · 프로젝트 디렉터)

프로젝트 전체를 조망하고 방향을 잡아주는 총괄 역할입니다.

- **PD 보고서** — 현재 프로젝트가 어디까지 왔는지, 다음에 뭘 해야 하는지 정리한 종합 리포트
- **마일스톤 추적표** — Q1(프리프로덕션) ~ Q4(폴리싱)까지 진행 상황 점검
- **크로스 역할 리뷰** — 각 팀원의 산출물이 서로 어긋나지 않는지 교차 검증
- **의사결정 기록** — "왜 이렇게 결정했는지" 근거와 다음 액션을 남기는 문서

### 체리 (PM · 프로젝트 매니저)

일정과 할 일을 체계적으로 쪼개고 관리하는 역할입니다.

- **WBS (작업 분해 구조)** — 큰 목표를 에픽 → 스토리 → 태스크 3단계로 잘게 쪼갠 목록
- **스프린트 계획서** — 2주 단위로 "이번에 뭘 할지" 구체적으로 짜놓은 계획
- **시스템 상세 스펙** — 전투, 천칭, 증강 등 각 시스템의 동작 규칙을 상세히 적은 기획서
- **리스크 레지스터** — 프로젝트에서 문제가 될 수 있는 위험 요소를 미리 정리한 문서

### 보스 (BD · 밸런싱 기획자)

게임의 숫자와 세계관 일관성을 잡아주는 역할입니다.

- **밸런싱 시뮬레이션 결과** — 천칭 수치, 증강 누적, 난이도 곡선 등을 수학적으로 검증한 보고서
- **세계관 정합성 보고서** — 타로 원전과 캐릭터 설정이 서로 맞는지 교차 검증한 문서
- **데이터 시트 스키마** — 캐릭터/스킬/증강/몬스터/보스 수치를 담는 JSON 구조 설계
- **밸런싱 최종 보고서** — 시뮬레이션 결과를 종합한 최종 판단 문서

### 송이 (AD · 아트 디렉터)

게임의 시각적 분위기를 총괄하고 일관성을 지키는 역할입니다.

- **비주얼 아이덴티티 가이드** — "아르카나는 이런 느낌이다"를 정의하는 아트 방향 문서
- **아트 스타일 체크리스트** — 모든 그림이 같은 톤을 유지하는지 확인하는 점검표
- **카드 레이아웃 표준** — 카드의 4개 레이어(배경 + 캐릭터 + 프레임 + 이름표) 규격 정의
- **비주얼 컨셉 이미지** — 아트 방향을 보여주는 참고 이미지 생성

### 아트 트리오 (CA · 캐릭터 아티스트)

캐릭터의 외형과 포즈를 디자인하는 3인 팀(유빈, 예빈, 주희)입니다.

- **캐릭터 디자인 시트** — 플레이어블 7종 + 보스 5종 각각의 외형·의상·소품 상세 설계
- **포즈 가이드** — 정방향(안정적/위엄) / 역방향(역동적/불안정) 두 가지 포즈 참조 자료
- **카드 레이아웃 검증 결과** — Unity Sprite 규격에 맞게 캐릭터가 카드 안에 잘 들어가는지 확인
- **캐릭터 컨셉 이미지** — 실루엣, 의상, 포즈를 보여주는 참고 이미지 생성

### 령 (BA · 배경 아티스트)

게임 세계의 공간과 분위기를 그려내는 역할입니다.

- **스테이지별 배경 컨셉 가이드** — 4개 스테이지 각각의 건축 양식·색감·분위기 정의
  - Stage 1 (The Chariot): 황금+파랑, 로마네스크 성벽
  - Stage 2 (Justice): 붉은+금색, 고딕 성당
  - Stage 3 (The Hermit): 회색+보라, 수도원 폐허
  - Stage 4 (The World): 무지개+어둠, 무너지는 신전
- **조명/분위기 가이드** — Unity 2D 조명(URP) 설정과 Parallax 레이어 구성 기준
- **배경 컨셉 이미지** — 스테이지 무드보드와 레이어 시각화 참고 이미지 생성

### 모션 듀오 (Animator · 2D 애니메이터)

캐릭터의 움직임과 컷신을 기획하는 2인 팀(가현, 예리)입니다.

- **모션 스펙 문서** — 캐릭터별 전투 모션(정방향/역방향/궁극기) 24fps 프레임 상세 설계
- **프레임 가이드** — 각 동작의 예비동작 → 본동작 → 후속동작 타이밍 분해표
- **컷신 시퀀스** — 프롤로그/스테이지 전환/엔딩 등 연출 장면의 카메라·동작·대사 구성

### 밍키 (UX · UI/UX 디자이너)

플레이어가 게임을 편하게 쓸 수 있도록 화면을 설계하는 역할입니다.

- **UX 플로우 다이어그램** — 게임 시작부터 엔딩까지 화면 이동 흐름도 (Mermaid 형식)
- **전투 HUD 와이어프레임** — 카드 6장, 파티 HP, 천칭 UI 등 전투 화면 배치 설계
- **UI 컴포넌트 스펙** — 파티 설정, 증강 선택, 튜토리얼 등 각 화면의 상세 규격
- **접근성 가이드** — 색약 대응, 폰트 가독성 등 누구나 쓸 수 있는 UI 기준

### 니니 (VA · VFX 아티스트)

화면의 특수효과와 타격감을 만들어내는 역할입니다.

- **이펙트 스펙 문서** — 정방향(맑은 빛) / 역방향(어두운 왜곡) 스킬별 파티클 상세 설계
- **파티클 가이드** — Unity Particle System 파라미터 표 (크기, 속도, 색상, 수명 등)
- **타격감 연출 설계서** — 히트스톱(잠깐 멈춤), 화면 흔들림, 번쩍임 연출 기준
- **환경 이펙트 설계** — 스테이지별 배경 파티클 (먼지, 안개, 에너지 등)

### 주디 (SD · 사운드 디렉터)

게임의 소리를 설계하여 감정적 몰입을 만드는 역할입니다. (PD와 동일 인물)

- **BGM 구성 계획표** — 6가지 상황별 배경음악 방향 설계
  - 메인 화면(오케스트라+합창), 맵 탐색(앰비언트), 일반 전투(퍼커시브+현악), 보스 전투(풀 오케스트라), 사건 노드(피아노), 엔딩(서정적 오케스트라)
- **SFX 디자인 문서** — 카드 드로우, 정방향/역방향 카드, 천칭 기울기, 시너지 발동 효과음 설계
- **사운드 에셋 카탈로그** — 모든 사운드 파일의 목록·상태(계획/제작중/완료) 관리

### 우디 (TD · 테크 디렉터)

아트와 프로그래밍 사이를 연결하고 기술 표준을 세우는 역할입니다.

- **기술 스택 문서** — Unity URP 2D + Spine + Addressables 기반 파이프라인 설계
- **렌더링 파이프라인 가이드** — 4개 레이어(배경/캐릭터/UI/이펙트) 렌더링 순서와 설정
- **성능 최적화 기준** — 드로우콜 100 이하, GC 최소화, 오브젝트 풀링 등 성능 목표
- **데이터 구조 스키마** — JSON 직렬화 방식과 ScriptableObject 구조 설계

### 진 (Programmer · 서브 프로그래머)

기획서의 규칙을 실제 동작하는 코드로 만드는 역할입니다.

- **시스템 구현 코드** — 전투 → 천칭 → 카드/시너지 → 궁극기 → 증강 순서로 C# 구현
- **아키텍처 설계서** — FSM(상태 기계), Observer(관찰자), Command(명령) 패턴 기반 구조
- **데이터 에셋** — ScriptableObject 기반 캐릭터/스킬/증강 데이터 설계
- **데이터 로더** — JSON/CSV 파일에서 게임 수치를 불러오는 시스템 (밸런싱 팀이 코드 수정 없이 수치 조정 가능)

### 보니 (QA · 품질보증)

극단적인 상황에서도 게임이 문제없이 돌아가는지 검증하는 역할입니다.

- **테스트 케이스 설계서** — 5대 영역(전투/천칭/증강/윤회/시너지) 검증 시나리오
- **QA 보고서** — 테스트 범위, 발견된 버그, 극단 케이스 결과 종합 리포트
- **버그 추적 양식** — 심각도(Critical/Major/Minor/Trivial) 분류별 버그 목록
- **PC 빌드 체크리스트** — 해상도, 프레임레이트, 한글 폰트, 저장/로드 동작 확인

## 3. 사용 도구

팀이 작업에 활용할 수 있는 도구들입니다.

### 이미지 생성 (Gemini 기반)

AI로 컨셉 아트 참고 이미지를 만들 수 있습니다.

- **사용하는 팀원:** 송이(AD), 아트트리오(CA), 령(BA)
- **모델 선택:**
  - **flash** (빠른 생성) — 초안, 무드보드, 색감 테스트 등 빠르게 확인할 때
  - **pro** (고품질) — 정교한 최종 컨셉 아트가 필요할 때
- **미리 준비된 프리셋:**
  - `character_concept` — 캐릭터 컨셉 (실루엣, 의상, 포즈)
  - `card_art` — 카드 일러스트 (프레임 안의 캐릭터)
  - `background_concept` — 배경 컨셉 (스테이지 환경)
  - `vfx_reference` — 이펙트 참고 (파티클, 빛 효과)
  - `worldview_concept` — 세계관 컨셉 (분위기, 톤)
- **결과물:** `output/design/` 폴더에 PNG 파일로 저장

### 영상 생성 (Veo 3.1 기반)

AI로 컷신이나 분위기 영상을 만들 수 있습니다.

- **기능:**
  - 새 영상 생성 (4초 / 6초 / 8초)
  - 기존 영상 연장 (라운드당 +7초)
  - 한 번에 1~4개 샘플 생성
- **설정 옵션:**
  - 화면 비율: 가로(16:9) 또는 세로(9:16)
  - 해상도: 720p 또는 1080p
  - 오디오 포함/제외 선택 가능
- **결과물:** `output/video/` 폴더에 MP4 파일로 저장

### 밸런싱 시뮬레이터

게임 수치의 균형을 수학적으로 검증하는 도구입니다.

- **사용하는 팀원:** 보스(BD), 보니(QA)
- **시뮬레이션 종류:**
  - 천칭 시스템 — 정방향/역방향 효과가 극단값(-3~+3칸)에서도 정상 작동하는지
  - 증강 누적 — 합연산+비율연산이 쌓였을 때 수치가 폭주하지 않는지
  - 난이도 곡선 — 스테이지가 진행될수록 적절한 난이도 상승인지
  - 시너지 발동률 — 세트 효과가 의도한 확률대로 발동하는지

- **생성 가능한 스키마:** 캐릭터(CharacterData), 스킬(CardData), 증강(AugmentData), 몬스터(MonsterData), 보스(BossData)
